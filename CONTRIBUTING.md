# Contributing

Thanks for your interest in improving these guidelines.

## How to Contribute

1. Open an issue describing the improvement or correction.
2. Provide clear examples and sources when possible.
3. Keep changes focused and consistent with the existing style.

## Style Guide

- Use clear, concise English.
- Prefer practical, framework-specific examples (FastAPI, Django).
- Keep lists actionable and avoid redundant advice.
- Avoid non-ASCII characters unless already used in the file.

## Submitting Changes

1. Create a branch.
2. Make your changes.
3. Run basic linting or checks if applicable.
4. Open a pull request with a brief summary.
